Identity Access Modes

The Azure Key Vault Provider offers four modes for accessing a Key Vault instance
Service Principal

Use a Service Principal

Pod Identity

Use Pod Identity

VMSS User Assigned Managed Identity

Allows Azure KeyVault to use the user assigned managed identity on the k8s cluster VMSS directly

VMSS System Assigned Managed Identity

Allows Azure KeyVault to use the system assigned managed identity on the k8s cluster VMSS directly
